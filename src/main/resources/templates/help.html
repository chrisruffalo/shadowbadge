General Overview

The Shadowbadge firmware and software are based around the Badgy device in a B/W/R (black, white, and red) screen configuration.

Controls

There is a five-way switch on the front of the Badgy that is used to give rudimentary input commands during operation. While we hope to expand this to an actual menu system this is enough to give basic input capabilities at this stage. The controls only apply to a Badgy device running the Shadowbadge firmware.

All control inputs are given facing the device with the five-way switch to the right of the screen. The controls can be used to perform a variety of actions at power-on. All directions/controls are based on the device being oriented as in the picture above.

During Power On
  • Up: Delete device configuration and start from scratch
  • Right: Forget current wifi settings
  • Center: Put the device into refresh mode immediately (pulls configuration from this web application)
  • Left: Put the device into OTA update mode to allow new firmware to be used

Or during the normal operation of the device.

While the Device is On
  • Up: Cycle icon (Red Hat, Shadowman)
  • Center: Put the device into refresh mode until an update happens

Getting the Firmware

You can download the current version of the Shadowbadge firmware for the Badge device from the downloads page. The version hosted there is intended for use with the current version of the Shadowbadge web application. This firmware is a binary file that needs to be loaded onto the Badgy device.

Loading the Firmware

If you are using the default Badgy firmware then you can enter the OTA (over the air) update mode by holding down the center button on the five-way switch as you slide the device from the "off" to the "on" position.

If you are using Shadowbadge firmware to enter OTA update mode the five-way switch needs to be held in the >left (toward the device's screen) position as the device is powered on.

Once you are in OTA mode you can connect to the access point provided by the Badgy's WiFi with the on-screen details. The Badgy device (in either firmware) will need to be connected to a wifi network in before you can connect to the update server.

Connecting to Wifi

When using any firmware (Badgy default or Shadowbadge) the device will remember the last AP that you connected it to. If it cannot find or connect to that AP it will show a screen with details for connecting to the device's AP which will provide you with a captive-portal webpage you can then use to connect the device to a local AP.

When using the Shadowbadge firmware the AP's SSID and password are generated so that you must be in view of the screen to connect. To make this process faster a QR code is displayed that some devices can scan to connect to the AP.

Once connected to the device's AP a captive-portal page will be displayed and you will be able to connect to the page and scan for local access points and configure the device's wifi.

Claiming the Badge

When the device is powered on for the first time with the Shadowbadge firmware it needs to be claimed (registered to a given user) before it can be configured. (Also, after a settings reset.) Before the device can be claimed it needs to connect to wifi.

When the device is connected to Wifi and ready to be claimed it will show a screen with a QR code that can be scanned in order to claim the badge. This will take you through the process to register with Shadowbadge (if you have not already) and then claim the badge for you.

Once the badge is claimed the screen will change to show the claim status. The name of the badge will be the same as the AP name during installation and will be shown in your browser and on the badge details screen.

Once you are on the badge details screen you can configure the badge to your specifications.

Configuring the Badge

The Shadowbadge web application provides a form for configuring various information that will be shown in the standard layout of the badge. This includes the icon, the header (your name), and other lines on the badge. The web UI will provide suggestions for these fields but feel free to be appropriately creative.

Unclaiming a Badge

In the Shadowbadge web application you have the ability to "unclaim" a badge. This is useful if you want to pass the badge to someone else. Once you click "unclaim" and then "unclaim" again the badge will be removed from your list of badges.

Next time the badge updates it will revert to claim mode and be ready to be scanned and claimed. This may require that you put the badge in "refresh" mode either by holding the center button and turing the unit on or, if the unit is already on, pressing the center button to trigger a refresh cycle.

Resetting a Badge

Holding "up" on the badge's five-way joystick will cause the badge to reset during boot. It will "forget" all of the wifi settings and the configuration it previously downloaded from the Shadowbadge server.

Power Optimization

While the Badgy device is power efficient the best way to conserve power is to allow the device to update and then turn it off. The screen will stay sharp for 24-48 hours or so before it fades some and needs to be updated again.
